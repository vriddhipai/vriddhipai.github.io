P2P File System

A P2P network resembles some features of the popular Bit-torrent Protocol, but much simplified. There are two pieces of software â€“ peer and file owner. The file owner has the file which it breaks into chunks of 100KB, each stored as a separate file. This file owner listens on a TCP port where the file owner is the server that can run multiple threads to serve multiple clients or peers simultaneously. I started by programming the the server to ensure the connection with the port was established to have it ready to listen to client requests. Once the server was up and running, I programmed the logic for a peer so it could connect to the file owner to download some chunks. I then programmed a peer to establish its two threads of control, one acting as a server that uploads the local chunks to another peer (referred to as upload neighbor), and the other acting as a client that downloads chunks from a third peer (referred to as download neighbor). So each peer has two neighbors, one of which will get chunks from one peer and the other will send chunks to another peer. This way, there's a direct path from any peer to any other peer, thus decentralizing the server's control during the distribution process.

NBA Player Database

NBA Player's Database is a web application project that I helped create as a team project in COP 5725, Spring 2019. The project helped me learn how to design and implement a dynamically responsive web site. Front-end pages are developed using PHP with Boostrap framework to set the UI template. Within the first two weeks, we found a large dataset on Kaggle with real NBA players' records and I got to design the website's look, feel, flow and control. The point of recreating such a website was to allow fans to have a comprehensive search experience when wanting to know the history of their favorite players and teams or have even intuitive forecast-based queries answered. For instance, users get to query details like the probability of the team that has the most total salary of players winning the championship and the probability of a random team winning the championship. Such queries show up as dynamic visualizations for a better experience for the user, which we facilitated through Chart.js (https://www.chartjs.org/), an open-source JavaScript library for data visualization. The queries were developed in SQL and the database was established on Oracle Database System. This project helped me not just gain experience with full-stack web application development but also explore my strenghts in the design and team leadership.

Check it out here: NBA Player Database

Gossip Protocol

Gossip Protocol is a communication protocol that works on randomization for infromation dissemination thus bringing much more reliability and robustness, especially when the nodes/participants involved need access to the data simulatenously. Inspired by one of my professors' meticulous research and expertise on the same, I utilized F# and Akka.NET to establish a network of nodes in a network requiring parallel access to some information or "gossip" and demonstrated how the Actor Model facilitates the dissemination when the underlying network's topology is varied. This helped me further understand how the gossip protocol behaves when external factors come in to play!

Check it out here: Gossip Protocol
